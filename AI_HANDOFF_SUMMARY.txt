Most important new code-level finding
- The next real bug was not another missing table. It was a broken live helper routine in bank 02:
  - `sub_973A` / `loc_9792`
  - file:
    C:\Users\Jake Diggity\Documents\GitHub\NES-TO-SEGA-GENESIS\src\banks\generated\bank_02_gen68k.asm
- Comparing the translated code against the NES original showed missing live operations in the healthy intro/manual lane:
  - missing `LDY ram_042B_enemy + $03` at original `02:974F`
  - missing second sprite-X write at `02:978B`
  - missing `STA ram_042B_enemy + $05` at `02:978F`
  - missing `INC ram_042B_enemy + $04` at `02:9792`
- Those were restored in the generated source-of-truth file.
- One assembler issue had to be corrected too:
  - `ram_pos_X_enemy+1(D1.L)` was not accepted by the 68000 backend
  - it was rewritten as:
    `MOVEA.L #ram_pos_X_enemy+1,A0`
    `MOVE.B  D0,(A0,D1.L)`

Why this helper bug matters more than the old table-only batches
- It is on the live healthy intro/manual path, not a speculative dead branch.
- It affects sprite/helper state used while the intro/story screen is being assembled.
- It is a concrete code divergence from the NES original, not just “maybe missing data.”
